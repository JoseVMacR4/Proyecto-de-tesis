Políticas de Uso Interno
1. Introducción y Aceptación

El presente documento establece las condiciones de uso del sistema "ETECSA Finanzas - Gestión Central", una herramienta interna desarrollada para la Unidad de Conciliación. Al acceder y utilizar este sistema, el usuario acepta cumplir con estas políticas. El acceso está estrictamente restringido al personal autorizado.

2. Uso del Sistema y Confidencialidad

El sistema tiene como propósito exclusivo la gestión de estados de cuenta bancarios, la conciliación de transacciones y la administración financiera interna. Toda la información procesada es de carácter estrictamente confidencial. Queda prohibida la extracción, divulgación o uso de estos datos para fines ajenos a las funciones laborales asignadas.

3. Roles y Control de Acceso

El acceso a las funcionalidades del sistema está regido por un esquema de permisos granulares basado en cuatro roles predefinidos:

  • Administrador: Acceso total al sistema, gestión de usuarios, entidades y configuraciones globales.
  • Analista Económico / Analista Comercial / Analista Financiero: Acceso restringido según los permisos asignados.

El usuario es responsable de mantener la confidencialidad de sus credenciales (usuario y contraseña). El uso de la cuenta es personal e intransferible.

4. Responsabilidades en el Procesamiento de Datos

Carga de Archivos: Los usuarios autorizados para subir archivos con estados de cuenta deben garantizar que los documentos provengan de fuentes bancarias oficiales y no hayan sido alterados.

Conciliación: Las acciones de marcar transacciones como conciliadas deben realizarse con la debida diligencia, ya que impactan directamente en las estadísticas globales de saldos y reportes de la Unidad.

5. Auditoría y Monitoreo

Para garantizar la seguridad y transparencia, el sistema cuenta con un módulo de auditoría permanente. Se registrarán de forma automática todas las actividades del usuario, incluyendo inicios y cierres de sesión, así como cualquier operación de creación, lectura, actualización o eliminación de entidades y transacciones.

6. Reporte de Incidencias

Cualquier falla en el sistema, error de datos o sospecha de brecha de seguridad debe ser notificada inmediatamente a través del Sistema de Tickets interno, categorizando adecuadamente el problema.

Políticas de Seguridad
1. Propósito y Alcance

Este documento detalla los mecanismos de seguridad implementados en el sistema "ETECSA Finanzas - Gestión Central" para proteger la integridad, confidencialidad y disponibilidad de los datos financieros procesados mediante su arquitectura en Django y PostgreSQL.

2. Autenticación y Autorización

Control de Acceso: Todas las vistas y endpoints críticos del sistema están protegidos mediante el decorador @login_required, impidiendo el acceso a usuarios no autenticados.

Gestión de Contraseñas: Las contraseñas de los usuarios no se almacenan en texto plano. Se utiliza el sistema de hashing criptográfico nativo de Django para almacenar credenciales seguras.

Manejo de Sesiones: El sistema gestiona las sesiones de usuario de forma segura en el backend, invalidando tokens al realizar logout o tras períodos de inactividad prolongada.

Permisos Granulares: El control de acceso basado en roles (RBAC) restringe la ejecución de acciones sensibles estrictamente a los perfiles autorizados.

3. Protección contra Vulnerabilidades Web

Protección CSRF: Todas las peticiones que modifican el estado del servidor, incluyendo la carga de estados de cuenta y la edición de transacciones, están protegidas mediante tokens Cross-Site Request Forgery (CSRF) nativos de Django.

Protección XSS e Inyección SQL: El uso del ORM de Django previene inherentemente los ataques de inyección SQL mediante la parametrización de consultas en la base de datos. Asimismo, el motor de plantillas Jinja2/Django Templates escapa automáticamente el contenido renderizado para prevenir ataques de Cross-Site Scripting (XSS).

4. Trazabilidad y Registro de Auditoría

El sistema implementa una trazabilidad estricta para cumplir con los estándares financieros:

Registro de Actividades: Se mantiene un log inmutable de las acciones de los usuarios, almacenando la acción realizada, el autor, la descripción detallada, metadatos y la marca de tiempo exacta (timestamp).

Trazabilidad de Entidades: Todos los registros críticos cuentan con campos de timestamps (fecha de creación y última modificación) para su seguimiento temporal.

5. Resiliencia y Manejo de Errores

Gestión de Excepciones: Los errores no manejados no exponen el código fuente ni los tracebacks en el entorno de producción.

Sistema de Tickets: Se provee un mecanismo interno estructurado para la gestión de vulnerabilidades y bugs. Los usuarios pueden levantar tickets categorizados que el equipo de administración puede gestionar, asegurando un ciclo de vida controlado para cualquier anomalía del sistema.
